Database Schema

This schema shows two core tables for managing donors and donations. IDs are auto-incremented, and relationships are enforced with foreign keys.

Donors Table

Column Name Data Type Key / Notes
donor_id INTEGER Primary Key, Auto-increment
first_name TEXT
last_name TEXT
email TEXT Unique
phone TEXT Optional
created_at DATETIME Timestamp for record

Donations Table

Column Name Data Type Key / Notes
donation_id INTEGER Primary Key, Auto-increment
donor_id INTEGER Foreign Key → Donors(donor_id)
amount DECIMAL
donation_date DATE
campaign TEXT Optional

Relationships

Each donor can have multiple donations (1-to-many relationship):
